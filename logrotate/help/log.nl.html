Rotatie schema
Dit bepaald hoe vaak de log word geroteerd.

Maximum grote voor rotatie
Zet een limiet op de grote van de log file voordat het word geroteerd. Dit veld kan worden gebruikt in plaats van het rotatie schema indien uw voorkeur uitgaat naar de grote in plaats van een op tijd gebaseerde limiet.

Aantal oude logs om te bewaren
Dit bepaald hoeveel oude log files u wilt bewaren. Deze zullen normaal gesproken dezelfde naam hebben als de originele log, echter met een .0, .1 en zo verder als toevoeging. Indien u een 0 invult, zullen er geen oude log files worden bewaard en dus zal de inhoud van de log file verloren gaan wanneer die word geroteerd.

Comprimeer oude log files?
Wanneer deze optie is aangezet, zullen oude logs worden gecomprimeerd met gzip na de rotatie om schijf ruimte te besparen.

Roteren zelfs als de log file leeg is?
Indien dit is aangezet, zal deze optie de logrotatie een log file laten roteren, zelfs als deze leeg is, het resultaat zal een oude kopie zijn die word opgeslagen waar niets in staat.

Log file opnieuw aanmaken na de rotatie?
Wanneer deze optie is aangezet, zal de log file opnieuw worden aangemaakt na de rotatie in plaats van met rust gelaten voor de server die naar deze file schrijft of opnieuw aanmaakt. Standaard zal de file opnieuw worden aangemaakt met de originele permissies en eigenaar, echter u kunt door middel van de velden naast de Ja optie alternatieve file permissies, gebruikers en groepen specificeren.

Oude geroteerde logs opslaan in
Wanneer dit veld is ingesteld naar Zelfde directory als log file, zullen de oude geroteerde kopieen van de log file worden opgeslagen in dezelfde directory als de originele log. De Directory optie kan worden gebruikt om een alternatieve locatie te kiezen - echter, het moet altijd op hetzelfde filesysteem staan als de log file.

Uit te voeren opdrachten na de rotatie
Wanneer deze tekst box is ingevuld, zal de ingevulde shell opdracht worden uitgevoerd nadat de lof is geroteerd. Veel servers moeten opnieuw worden gestart als hun log word hernoemd zodat zij netjes naar de nieuwe file kunnen schrijven. In dat geval moet u een opdracht geven zoals killall -HUP httpd om een herstart te triggeren na een rotatie.
