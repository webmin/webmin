Root Toegang
Dit controleert op welke hosts de server de root gebruiker als root zal accepteren. Indien een client niet in de lijst voorkomt, zal deze zelfs als die de rechten heeft om te kunnen mounten als alleen-lezen of lezen-schrijven de root gebruiker behandelen als een Anonieme Gebruiker.
Hosts kunnen als volgt worden gespecificeerd :

Hostnaam of IP adres (zoals ftp.foo.com of 1.2.3.4)
Komt overeen met de host met de opgegeven naam of adres
Netgroep (zoals engineering)
Komt overeen met de host die lid is van de netgroep
DNS domein (zoals .foo.com)
Komt overeen met iedere host in het domein
Netwerk (zoals @10.254.1)
Komt overeen met iedere host in het netwerk
Netwerk/netmask (zoals @10.254.1/24)
Komt overeen met iedere host in het netwerk

Als extra kunt u bovenstaande notaties een toevoeging geven met een -, wat betekend dat de host, netgroep, domein of netwerk het niet is toegestaan om deze directory's of files te mounten. Dit kan handig zijn indien u een aantal hosts toegang wilt verlenen en een enkel lid wilt weigeren.
